Clasificación supervisada IV: Inducción de reglas y selección de características

Inducción de reglas

La extracción y aplicación de un conjunto de reglas es uno de los métodos de clasificación supervisada más transparentes, además de fáciles de utilizar y de entender. Los sistemas expertos se construyen a partir de la explicitación de una serie de reglas recogidas a partir del conocimiento de un experto en un dominio determinado, y los sistemas de inducción de reglas tratan de extraer estas reglas de forma automática. Una regla suele tener la expresión:

If (Xj=xj & Xi=Xi & ... & Xk=xk) Then C=c

La parte afectada por la condición se denomina antecedente de la regla (IF [antecedente]), mientras que el efecto de la activación positiva de la condición es el consecuente (THEN [consecuente]).

Los sistemas basados en reglas son aún más generales que los árboles de clasificación, pues éstos pueden ser siempre transformados en un sistema de inducción de reglas, mientras que el contrario no es siempre cierto.

Algoritmo 1R

Este algoritmo descansa en el hecho que un conjunto de reglas muy simples usualmente funcionan bien. Así, las 1-reglas clasifican una instancia de acuerdo a una única variable.

Para variables continuas, los algoritmos 1R discretizan el rango de valores en una serie de intervalos disjuntos a partir de la única restricción de que todos los intervalos deben contener más de un número predefinido de instancias con el mismo valor de clase.

Los valores faltantes de un atributo se considerarán como nuevos valores, tanto para variables continuas como discretas. Por cada posible valor xix_ixi​﻿ de la variable XiX_iXi​﻿, el algoritmo 1R construye una regla seleccionando el valor óptimo de clase (el más frecuente) como su consecuente.

Este algoritmo sólo extrae una regla por variable, que añade a un conjunto de posibles hipótesis y, posteriormente, selecciona aquella que tenga el mejor rendimiento sobre el set de entrenamiento de entre el conjunto de hipótesis.

Algoritmo AQR

EL algoritmo de generación de reglas AQ (AQR) es un sistema de inducción que usa el algoritmo AQ, precursor de la inducción de reglas. AQR genera un set de reglas de clasificación para cada clase. Cada una de estas reglas es del tipo IF [cover] THEN [class], donde un cover representa una combinación de tests sobre las variables predictoras por medio de disjunciones y conjunciones.

El test más básico de variable es un selector, y toman formas del tipo:

  • Xi=aX_i=aXi​=a﻿
  • Xi=b,cX_i=b,cXi​=b,c﻿
  • Xi<4X_i<4Xi​<4﻿

Una conjunción de selectors se llama complex, como por ejemplo Xi=a AND Xj>bX_i=a \textrm{ AND }X_j>bXi​=a AND Xj​>b﻿

Finalmente, un cover es una disjunción de complexes: (Xi=a AND Xj>b) OR (Xk=c,d AND Xj≥d)(X_i=a \textrm{ AND }X_j>b) \textrm{ OR } (X_k=c,d \textrm{ AND }X_j\geq d)(Xi​=a AND Xj​>b) OR (Xk​=c,d AND Xj​≥d)﻿

Cualquiera de estas expresiones (un selector, un complex o un cover) se dice que cubre una instancia si es cierta para ésta. Cada cover generado por el algoritmo se almacena con un valor de la variable de clase, que es aquel más comun para los elementos del set de entrenamiento que cubre.

El algoritmo AQR comienza escogiendo un valor de clase, y seguidamente, forma un cover que sirva como antecedente para la regla que determinará que una instancia tiene tal valor de la variable de clase. Para ello procede generado un complex, eliminando las variables que cubre del set de datos, y repitiendo este paso hasta que se han cubierto todas las instancias de la clase escogida, momento en el cual se obtiene el cover para tal clase.

Tras realizar este procedimiento para cada clase posible, finaliza el algoritmo.

El algoritmo AQR también realiza reducción de los antecedentes de las reglas de un complex siguiendo un proceso de maximización de instancias positivas cubiertas y minimización de instancias negativas excluídas. Para escoger el mejor complex para una clase dada, se sigue exclusivamente la maximización de instancias positivas cubiertas. En caso de duda, en el algoritmo AQR siempre se prefieren los complexes con pocos selectores.

Algoritmo CN2

Este algoritmo trabaja de forma iterativa, donde cada iteración busca un cover que cubra un gran número de instancias de una clase en particular, incluso si cubre algunas de otras clases. Cuando encuentra un buen cover, elimina las instancias que cubre del set de datos de entrenamiento y añade la regla resultante a una lista de reglas. Cuando ya no se pueden encontrar más covers satisfactorios finaliza el algoritmo.

Para encontrar cada cover, CN2 lleva a cabo una búsqueda general-a-específico podada. En cada paso de la búsqueda, CN2 retiene un set de tamaño limitado de los mejores covers encontrados hasta el momento y considera especializaciones de éstos para avanzar en la búsqueda.

Una especialización de un cover se alcanza añadiendo un nuevo término conjuntivo o eliminando un elemento disjuntivo en uno de sus complexes. CN2 genera y evalúa todas las posibles especializaciones de cada cover.

Algoritmo IREP

EL Incremental Reduced Error Pruning es un algoritmo de inducción de reglas en el que los datos se dividen en un set de crecimiento (growing set) y un set de poda (pruning set).

El objetivo es formar un set de reglas inicial que sobreajuste (overfits) el set de crecimiento siguiendo cualquier método heurístico de forma voraz, añadiendo las reglas una a una y eliminando las instancias que cubre una vez se encuentra una nueva regla. Cuando no hay más instancias positivas o la regla encontrada tiene un ratio de error demasiado alto, se simplifica el set de reglas de forma repetitiva aplicando uno de entre un set de operadores de poda.

Se elige aquel operador que produzca la mayor reducción del error estimado sobre el set de poda (los más típicos son la eliminación de conjunciones o de reglas completas). La simplificación finaliza cuando aplicar un operador de poda aumentaría el error de clasificación en el set de poda.

Para construir una regla primero se particionan las instancias positivas y negativas sobre una clase de forma aleatoria en subconjuntos de crecimiento y de poda, dando lugar a cuatro subconjutos:

  • GrowPos: Instancias positivas usadas para el crecimiento
  • GrowNeg: Instancias negativas usadas para el crecimiento
  • PrunePos: Instancias positivas usadas para la poda
  • PruneNeg: Instancias negativas usadas para la poda

Algoritmo RIPPER

El algoritmo Repeated Incremental Pruning to Produce Error Reduction mejora el algoritmo IREP mediante la adición de una serie de cambios frente a éste:

  • Una nueva métrica en lugar de vvv﻿ para la fase de corte
  • Una heurística distinta para determinar cuándo se debe de parar de añadir reglas a un set de éstas
  • Un post-pase sobre el conjunto de reglas que lo mejora

Selección de subconjuntos de características

Los mecanismos de selección de subconjuntos de características (Feature Subset Selection, FSS) permiten identificar y eliminar las variables redundantes. Su uso conlleva una serie de ventajas:

  • Reducción de la dimensionalidad de los datos
  • Hacen que los algoritmos de aprendizaje aprendan más rápido y eficientemente
  • Mejoran el acierto de los clasificadores
  • Ayudan a la interpretación del modelo aprendido

Métodos filter de FSS

En los algoritmos de tipo filter, la relevancia de una característica (filter univariante) o de un subconjunto de éstas (filter multivariante) se determina atendiendo exclusivamente a propiedades intrínsecas de los datos y las variables que los describen. Por sus características son:

  • Escalables a conjuntos de datos de alta dimensionalidad
  • Rápidos y simples a nivel computacional
  • Propensos a ayudar con problemas de overfitting
  • Independientes del algoritmo de clasificación

Además, sólo necesitan ser ejecutados una vez.

Los métodos de filter univariantes toman las variables originales X1,...,XnX_1,...,X_nX1​,...,Xn​﻿ y las ordenan de acuerdo a una función de evaluación f(x)f(x)f(x)﻿. Seguidamente, la aplicación de un umbral de puntuación permite seleccionar las sss﻿ mejores variables del ranking, que constituirán el subconjunto de características sobre el que se ejecutará el algoritmo de ML.

Los métodos de filter multivariantes no evalúan cada característica independientemente, sino que evalúan subconjuntos SSS﻿ de éstas que haya encontrado el algoritmo previamente. El mejor subconjunto, S∗S^*S∗﻿, se halla por medio de un problema de optimización empleando la función de evaluación f(S)f(S)f(S)﻿ y será el que se utilice para el aprendizaje del modelo de ML.

Para realizar el entrenamiento y pruena del modelo con los métodos de filter a menudo se emplea un esquema hold-out, dividiendo el set de datos original en:

  • Un conjunto de entrenamiento sobre el que se aplicará en primer lugar un procedimiento de selección de características por filter y que, tras entrenar el modelo de ML proporcionará el clasificador ϕtraining\phi_\textrm{training}ϕtraining​﻿
  • Un conjunto de prueba sobre el que se probará el modelo y permitirá estimar el rendimiento del clasificador

Para clasificadores continuos

Hasta ahora, los métodos de filter univariante paramétrico sólo habían considerado modelos de predicción discreta. No obstante, también hay métricas para variables predictoras continuas.

Para problemas de clasificación binaria la elección más popular es el test-t de Welch, que se puede utilizar con el fin de comparar cada característica, de forma análoga al uso del test de Chi cuadrado para estimar el poder discriminativo que tiene una característica sobre cada clase.

En problemas de clasificación no binaria se utiliza el test ANOVA unidireccional con medidas repetidas.

Tests de Mann-Whitney y de Kruskal-Wallis

El test de Mann-Whitney es un método para determinar la igualdad en las medias poblacionales de dos muestras. Para ello se supone inicialmente que las observaciones de ambos grupos son independientes, la hipótesis nula, bajo la cual, la distribución de partida de ambos grupos debe ser la misma. No obstante, si las muestras no fueran independientes, entonces la media muestral de una de las poblaciones debe exceder la de la otra. Para realizar este test se emplea el p-valor de las características, siendo las más adecuadas para incluirse en el modelo clasificador aquellas que presenten el p-valor más pequeño.

El test de Kruskal-Wallis funciona de forma similar pero permite comprobar más de dos poblaciones al mismo tiempo (se puede utilizar en clasificación no binaria).

Ramificación y poda (branch and bound)

Este método se basa en la creación de un árbol con todas las posibilidades a explorar y la posterior poda de algunas ramas que se consideran sub-óptimas y no se explorarán, reduciendo así el espacio de búsqueda.

Para la construcción del árbol se fija el número máximo nodos hijo y se parte de un nodo raíz que representa el conjunto con todas las características. Cada nodo hijo se forma eliminando una de las variables predictoras, lo que debe hacerse de forma ordenada para no repetir nodos hoja. Cada nodo hoja lleva codificada una configuración del subconjunto de características escogida por eliminación de las variables en el camino que lleva a él.

Seguidamente se opera por niveles del árbol, evaluando los subconjuntos resultantes en un nodo y podando toda su descendencia si no satisfacen un criterio de suboptimalidad, f(Snodo)<Bf(S_\textrm{nodo})<Bf(Snodo​)<B﻿.

Otros métodos metaheurísticos

También se puede explorar inteligentemente el espacio de búsqueda empleando metaheurísticas como:

  • Optimización por colonias de hormigas, inspirada en cómo las hormigas encuentran el camino más corto entre su hormiguero y la comida empleando rastros de feromonas que se refuerzan con su paso. Cada hormiga elige un camino según una regla basada en la concentración de feromonas, favoreciendo estadísticamente que el camino más corto tenga la mayor concentración de feromonas (pues es el que recorren más hormigas por unidad de tiempo).
  • Algoritmos evolutivos y genéticos, que trabajan con una población de individuos sujeta en cada generación a la reproducción, mutación y evaluación para el posterior reemplazo de la población anterior con los mejores nuevos individuos.
  • Algoritmos de estimación de la distribución, que tratan de hallar soluciones óptimas buscando la función que gobierna la distribución de las mejores entre éstas. Para ello parten de una población inicial con un cierto número MMM﻿ de individuos, cada uno en representación de una potencial solcución. En cada iteración se seleccionan los N<MN<MN<M﻿ mejores individuos y se estima su función de distribución, para posteriormente generar una nueva población de MMM﻿ individuos de acuerdo con ésta. El proceso se repite hasta que se cumple el criterio de parada.
