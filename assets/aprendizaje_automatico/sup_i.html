Clasificación supervisada I: Métodos de evaluación

Un problema de clasificación supervisada requiere de tres componentes:

  • Un espacio de muestras (o de instancias) en el que se encuentran las características de los datos, ΩX\Omega_XΩX​﻿:
    • Los datos son muestras aleatorias en forma de vector x=(x1,...,xn)∈Rn\textbf x=(x_1,...,x_n)\in \mathbb R^nx=(x1​,...,xn​)∈Rn﻿tomadas independientemente de una distribución de probabilidad fija pero desconocida p(x)p(\mathbf x)p(x)﻿.
    • La componente i−i-i−﻿ésima del vector x\mathbf xx﻿, xix_ixi​﻿, se toma aleatoriamente del subespacio de muestra ΩXi\Omega_{X_i}ΩXi​​﻿ y contiene el valor de la variable predictora XiX_iXi​﻿ para esa instancia específica
    • El espacio muestral resulta del producto tensorial de los subespacios muestrales correspondientes a los valores disponibles para cada variable predictora:
  • Un espacio de etiquetas, que contiene las diferentes clases que pueden llevar asociadas las instancias de variables predictoras:
    • Las etiquetas, ccc﻿, se obtienen de una variable aleatoria CCC﻿
    • La distribución de probabilidad condicional p(c∣x)p(c|\textbf x)p(c∣x)﻿ dado un vector del espacio de instancias, así como la distribución de probabilidad conjunta p(x,c)p(\textbf x,c)p(x,c)﻿ de casos (cada caso corresponde a una instancia con su etiqueta) son desconocidas
  • Un algoritmo de aprendizaje que implementa un conjunto de funciones sobre el espacio de instancias y cuya salida pertenece al espacio de etiquetas. La aplicación de este algoritmo a un set de datos de instancias clasificadas D={(x1,c1),...,(xN,cN)}\mathcal D=\{(\mathbf x^1, c^1),...,(\mathbf x^N,c^N)\}D={(x1,c1),...,(xN,cN)}﻿ proporcionará un modelo de clasificación ϕ\phiϕ﻿

El modelo de clasificación obtenido realiza una partición (división en conjuntos disjuntos que cubren completamente un espacio) del espacio de instancias en una serie de regiones de decisión. En particular, existirá una región de decisión por etiqueta, de forma que si x\mathbf xx﻿ cae en la región asociada con ccc﻿, el modelo de clasificación asignará ϕ(x)=c\phi (\mathbf x)=cϕ(x)=c﻿.

Las regiones están separadas por fronteras de decisión, hipersuperficies en el espacio correspondientes a instancias que el clasificador asigna dos clases diferentes con la misma certeza.

Cuanto más flexibles sean las fronteras del clasificador, mejor será su rendimiento.

Curva ROC

La curva ROC, acrónimo de Característica Operativa del Receptor (Receiver Operating Characteristic) es una representación gráfica que permite comprobar cómo varia el rendimiento de un clasificador binario al variar el umbral de discriminación: el valor límite para la probabilidad a posteriori pϕ(c,x)p_{\phi}(c,\textbf x)pϕ​(c,x)﻿ para el que la clase predicha se considera +++﻿. Cada valor del umbral de discriminación proporciona un punto en la curva ROC.

Este gráfico se crea pintando en el eje yyy﻿ el TPR (true positive rate) frente al FPR (false postive rate) en el eje xxx﻿, y la curva ROC se obtiene uniendo los diferentes puntos en el espacio ROC, que es un cuadrado de área unidad, pues 0≤FPR≤10\leq \textrm{FPR} \leq 10≤FPR≤1﻿ y 0≤TPR≤10\leq \textrm{TPR} \leq 10≤TPR≤1﻿.

El punto (0,0)(0,0)(0,0)﻿ tiene FPR=TPR=0\textrm{FPR}=\textrm{TPR}=0FPR=TPR=0﻿, luego clasifica todas las instancias como negativas y el punto (1,1)(1,1)(1,1)﻿ tiene FPR=TPR=1\textrm{FPR}=\textrm{TPR}=1FPR=TPR=1﻿, luego clasifica todas las instancias como positivas. Los puntos en la diagonal FPR=TPR\textrm{FPR}=\textrm{TPR}FPR=TPR﻿ corresponden a los clasificadores aleatorios, y cumplen que la probabilidad de que el clasificador asigne la clase +++﻿ o −-−﻿ a un ejemplo es la misma. Todos aquellos puntos sobre la diagonal representan un clasificador que funciona mejor que uno aleatorio, mientras que los que estén por debajo funcionarán peor.

En general un clasificador con (FPR1,TPR1)(\textrm{FPR}_1,\textrm{TPR}_1)(FPR1​,TPR1​)﻿ es mejor que uno con (FPR2,TPR2)(\textrm{FPR}_2,\textrm{TPR}_2)(FPR2​,TPR2​)﻿ siempre que el primero se encuentre más a la izquierda y más arriba que el segundo en el espacio ROC, por lo que el mejor clasificador posible es el (0,1)(0,1)(0,1)﻿, pues clasifica todos los ejemplos correctamente, y el peor posible es el dado por (1,0)(1,0)(1,0)﻿, pues clasifica todos los ejemplos incorrectamente.

Para generar la curva ROC a partir de un clasificador con salida ϕ(xi)\phi(\textrm x^i)ϕ(xi)﻿ sobre un set de casos D\mathcal DD﻿, se toman los valores máximo (max\textrm{max}max﻿) y mínimo (min\textrm{min}min﻿) de ϕ(xi)\phi(\textrm x^i)ϕ(xi)﻿ y se calcula el incremento a utilizar, incr\textrm{incr}incr﻿, como la mínima diferencia existente entre dos valores de la salida.

Los valores de umbral ttt﻿ sobre los que se probará el clasificador serán, por tanto t=min,min+incr,min+2⋅incr,...,maxt= \textrm{min},\textrm{min}+ \textrm{incr},\textrm{min}+ 2 \cdot \textrm{incr},...,\textrm{max}t=min,min+incr,min+2⋅incr,...,max﻿. A partir de estos valores se ejecuta el pseudocódigo:

// Input: Un clasificador phi() y constantes max, min, incr, N+ y N-
// Output: La curva ROC de phi()

for t=min to t=max in steps of incr
	TP=0
	FP=0
	for x in Dataset do:
		if phi(x)>=t:
			if x == +:
				TP=TP+1
			else if x == -:

			endif
		endif
	endfor

	FPR = FP/N-
	Add (TPR, FPR) to ROC curve
endfor
