Regresión logística

Mediante un clasificador de regresión logística se busca la probabilidad de que una instancia dada lleve asociada una determinada clase, con el fin de elegir como predicción para la instancia aquella clase que maximice la probabilidad a posteriori, es decir, argcmaxP(C=c∣x)\textrm{arg}_c\textrm{max}P(C=c|\mathbf x)argc​maxP(C=c∣x)﻿.

Los métodos de regresión logística describen la relación entre una variable de respuesta binaria YYY﻿, que se considera dependiente, y una serie de variables explicatorias o predictoras, que se consideran independientes. Su objetivo es determinar y medir la ausencia o existencia de una cierta relación entre una o más de las variables independientes con la dependiente. Las relaciones identificadas se emplean con el fin de predecir la probabilidad de que la variable de respuesta tome cada una de sus dos posibles clases, probabilidades que posteriormente se utilizarán para clasificar observaciones futuras.

  • Tener media 0
  • Tener varianza constante
  • Ser independiente

La regresión logística, además, es una técnica muy general, pues permite el uso de variables explicatorias continuas y categóricas y no precisa la suposición de hipótesis sobre las distribuciones.

Árboles de clasificación

Los árboles de decisión son una de las maneras más ampliamente utilizadas para aproximar funciones de valor discreto. Son muy robustos ante datos ruidosos y pueden ser re-representados como conjuntos de reglas de tipo if-then.

Los nodos terminales en un árbol de clasificación se denominan hojas (leaves), y proporcionan la clasificación de la instancia. Aquellos nodos que no son hojas especifican un test o prueba sobre un atributo (característica) de la instancia, de modo que las instancias no vistas (aquellas sobre las que no ha entrenado el árbol) se clasifican recorriendo el árbol desde la raíz hacia las hojas, testeando los atributos especificados en cada nodo hasta que se alcanza el nodo hoja que da su clasificación.

Un ejemplo muy sencillo de árbol es el siguiente, que permite clasificar una mañana de sábado según sea o no apropiada para jugar al tenis:

Un árbol representa una disjunción de conjunciones, es decir, una suma lógica (OR) de enunciados conjuntivos (AND). Cada camino en el árbol desde el nodo raíz hasta una hoja es una disjunción, de modo que el árbol corresponde a la unión por suma de todos los caminos.

Los problemas apropiados para el aprendizaje de árboles de decisión son:

  • Aquellos cuyas instancias vienen representadas por pares atributo-valor
  • Los que tienen una función objetivo con salida discreta (clasificación), aunque los árboles se pueden extender para ser usados con valores reales (árboles de regresión)
  • Los que necesiten robustez frente a errores en los datos de entrenamiento (ya sea en la clase o en los atributos) o que tengan datos faltantes en los atributos
  • Dominios de clasificación complejos en los que no haya una separación lineal clara

Algoritmo ID3

El algoritmo ID3 (Iterative dichotomiser) es una estrategia de búsqueda voraz por el espacio de todos los posibles árboles de clasificación que trata de construir el árbol desde la raíz hacia las hojas (lo que se conoce como top-down).

El algoritmo comienza, por tanto, estableciendo qué atributo debería testearse en el nodo raíz, pregunta que responde evaluando todos los atributos disponibles para determinar cómo de bien clasifican los ejemplos de entrenamiento. Tras seleccionar el mejor atributo, se crea un hijo de este nodo por cada posible valor del atributo seleccionado y se reparten los ejemplos de entrenamiento para cada descendiente.

Seguidamente se repite este proceso con cada atributo hasta que el árbol clasifica correctamente los ejemplos o hasta que no quedan atributos, momento en el que se etiqueta cada nodo hoja con la clase de los ejemplos que le corresponde. Así, cada atributo aparece como máximo una vez en cada camino.

La complejidad de los árboles creados por el método ID3 crece linealmente con el número de ejemplos de entrenamiento y exponencialmente con el número de atributos. Además, como usa propiedades estadísticas de todos los ejemplos, es poco sensible a errores en los datos.

Una extensión interesante del algoritmo ID3 consiste en modificar el criterio de parada, permitiéndole crear nodos hoja que no ajusten perfectamente los datos, sino que sólo escojan una etiqueta mayoritaria.

Como puntos negativos de este algoritmo se deben destacar dos:

  • Al ser voraz puede llegar a converger a óptimos locales
  • Favorece la selección de variables con muchos valores

Mejoras del algoritmo ID3

Los principales problemas del uso del ID3 son:

  • No es trivial determinar cuánto debe crecer el árbol (problema del overfitting)
  • Hay que elegir apropiadamente la métrica de selección de características
  • Hay que escoger una métrica para la selección de atributos apropiada
  • A priori no puede trabajar con:
    • Atributos continuos
    • Datos a los que le falten ciertos atributos
    • Atributos con diferentes costes

Problema del overfitting

Si se deja crecer un árbol hasta que sea lo suficientemente profundo como para clasificar todos los ejemplos pueden suceder dos cosas:

  • Si hay ruido en los datos lo puede aprender el árbol
  • Si hay sólo unos pocos ejemplos asociados con cada hoja es dificil producir una muestra significativa de la verdadera función objetivo

Cuando esto sucede el modelo no generaliza correctamente:

Para evitar el overfitting hay dos tipos de estrategia de simplificación del árbol:

  • Pre-poda (pre-pruning): Se deja de hacer crecer el árbol antes de que aprenda a clasificar todos los datos de forma perfecta. Para decidir si expandir un nodo particular va a produicir un aumento del rendimiento más allá del que tenga en el set de entrenamiento se utilizan tests estadísticos (como el chi cuadrado).
  • Post-poda (post-pruning): Se deja que el árbol aprenda a clasificar todos los datos de forma perfecta y luego se reemplazan subárboles por nodos hoja. Este método es más costoso, pero tiene mejor rendimiento en la práctica, pues en las estrategias pre-poda no es trivial estimar cuándo dejar de hacer crecer el árbol. Se realiza desde los nodos hoja hacia la raíz (down-top).

    Para comprobar si se debe podar un subárbol, es decir, sustituírlo por uno nodo hoja cuya etiqueta asociada sea la mayoritaria, se emplea un set de ejemplos dedicado (set de prueba para la poda), y se efectúa la poda si el árbol resultante funciona al menos tan bein como el original. Iterativamente, se poda el nodo cuya eliminación mejore más la precisión hasta que el siguiente paso de poda sea detrimental para el rendimiento del árbol.

Árboles con atributos continuos

La opción usual para trabajar con atributos continuos en un árbol de decisión consiste en discretizarlos, es decir, en particionarlos en un set discreto y finito de intervalos. De ser discretizados en dos intervalos, uno debe seleccionar un umbral ccc﻿ que separe los dos posibles valores discretos de la nueva característica.

La elección de ccc﻿ debe hacerse de forma tal que produzca la máxima ganancia de información. Este valor debe caer en la región en la que cambia el valor de clase de las instancias una vez ordenadas de acuerdo al atributo continuo.

Para refinar el proceso de discretización de una variable, éste puede hacerse de forma dinámica. Cuando debe evaluarse un atributo continuo AAA﻿ frente a una serie de atributos discretos para ser añadido al gráfico:

  1. Se ordenan los ejemplos de acuerdo a su valor AAA﻿
  1. Se toman los valores de AAA﻿ de dos ejemplos adyacentes con diferente etiqueta de clase y, o bien se promedian para ser ccc﻿ o se evalúan según la ganancia de información asociada con cada uno
  1. Con este nuevo ccc﻿, se discretiza la variable y se compara con los otros atributos discretos

Así, si A AA﻿ no es escogido en esta ocasión, puede ser que ccc﻿ sea diferente cuando se vuelva a evaluar (porque cambien los ejemplos disponibles).

Otras alternativas consisten en utilizar más de dos intervalos o incluso combinaciones lineales de un conjunto de variables continuas para discretizarlas juntas.

También puede usarse este proceso para agrupar valores de una variable discreta que disponga de muchos valores posibles.

Árboles con atributos faltantes en los datos

La opción más sencilla cuando no se dispone de un atributo sería eliminar la instancia incompleta, pero este caso supone el descarte de información aprovechable. Para evitar esta situación, se puede estimar el valor de los atributos faltantes mediante un mecanismo de imputación. Los más populares son:

  • Asignar al ejemplo la moda, es decir, el valor más comun de la clase. Éste se puede calcular a su vez:
    • De entre los ejemplos de entrenamiento asociados al nodo del que falta el atributo
    • De entre los ejemplos de entrenamiento asociados al nodo que además tengan la misma etiqueta que aquel al que le falta el atributo
  • Asignando una distribución de probabilidad de cada posible atributo estimada a partir de las frecuencias del valor del atributo observadas en el nodo del que falta el atributo

Árboles con atributos de diferente coste

Es recomendable favorecer que aquellos atributos que tengan coste más bajo se escojan en los nodos de inicio, es decir, aquellos cercanos a la raíz. Para ello se debe utilizar una métrica de selección de características que tenga en cuenta el coste del atributo, como por ejemplo: I(C,Xi)cost(Xi)\frac{\mathbb I(C,X_i)}{\textrm{cost}(X_i)}cost(Xi​)I(C,Xi​)​﻿.

De este modo, confiando en los atributos de coste más elevado (aquellos más discriminativos) para cuando se necesite precisar la clasificación más cerca de las hojas, se logrará una clasificación más estable y, de cara a las aplicaciones prácticas, a menudo menos costosa y arriesgada.

Árboles para variables de clase continua

Los árboles de decisión pueden generalizarse para ser aplicados a funciones de regresión. En este caso la variable de clase es continua, con lo que deben realizar una predicción numérica. Se pueden seguir dos estrategias para conseguir árboles de decisión con variables de clase continua:

  • Árboles de regresión: El valor de los nodos hoja representa la media del valor de la respuesta de las instancias que alcanzaron cada hoja. Esta clase de árboles puede trabajar con cualquier tipo de atributos, y permite aproximar funciones no lineales como funciones constantes por trozos.
  • Árboles de modelo: En las hojas hay un modelo de regresión lineal para predecir la respuesta de las instancias que llegan a ella (en este sentido los árboles de regresión son un caso particular de árbol modelo). Funciona con todo tipo de variables atributo, pero aún mejor con las continuas.

La diferencia entre ambos se puede apreciar confrontando los árboles a los que dan lugar aplicados sobre el mismo modelo de regresión. El árbol modelo dispone de un modelo lineal para la predicción (LMLMLM﻿) en cada hoja.
