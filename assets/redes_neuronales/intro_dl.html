Evaluación del rendimiento de un modelo

Entrenamiento y prueba

Antes de entrenar el modelo hay que preparar el set de datos. Los primeros pasos son:

  • Los datos pueden provenir de fuentes diferentes, así que hay que integrarlos todos correctamente, uniformizándolos
  • Hay que lipiar los datos y prepararlos en el formato adecuado
  • Se puede llevar a cabo una visualización de los datos para extraer información de ellos que se empleará para efectuar un análisis de reducción de dimensionalidad
  • Si los hay, se deben resolver los problemas éticos que implique el set de datos (por ejemplo, a menudo es necesario ejecutar un proceso de anonimización)

Posteriormente, hay que dividir el set de datos disponible en un subconjunto destinado al entrenamiento (que debe ser el más grande), uno de desarrollo y, si se desea, uno de prueba, procurando que todos los subconjuntos estén constituídos sobre una distribución lo más parecida posible.

Cada uno de estos conjuntos de datos tiene un propósito diferente. En el diagrama incluído más abajo viene representado el flujo de ajuste de un modelo neuronal. Comienza en la flecha punteada marcada como inicio tras configurar el modelo y sus hiperparámetros (a priori debe hacerse sin conocimiento previo sobre el funcionamiento de la red) y posteriormente se ejecuta el entrenamiento sobre el modelo (utilizando el set de entrenamiento), obteniendo el conjunto de parámetros de la red. Tras este proceso se prueba el modelo obtenido compresivo de los parámetros calculados durante el entrenamiento y los hiperparámetros configurados al inicio sobre el set de desarrollo, lo que permite calcular la eficacia del modelo en términos de su nivel de acierto, su parcialidad (bias) o su varianza, y recomienza el ciclo. Cuando se determina que el error es suficientemente bajo, en lugar de probar el modelo sobre el set de desarrollo, se prueba sobre el set de prueba final y se obtiene el rendimiento conclusivo del modelo a partir de éste.

El proceso de ajuste de los hiperparámetros se debe realizar de forma manual (a diferencia de los procesos de entrenamiento y prueba del modelo), por lo que involucra un método de prueba y error iteracional hasta que se afina el modelo en el grado deseado, siendo esta una de las labores del ingeniero de DL.

Interpretación de Varianza y Bias

Cuando un modelo tiene alto bias, ocurre que no ajusta bien el set de entrenamiento, pues no está percibiendo la forma correcta subyacente de la función de distribución de los ejemplos, mientras que cuando tiene alta varianza puede estar sujeto a la interpretación del ruido en el set de entrenamiento como parte de esta función de distribución y, por tanto, no generalizará bien.

En general hay que compensar encontrando un equilibrio entre ambas medidas, ajustando los hiperparámetros para reducir el bias cuando el modelo no ajusta apropiadamente el set de entrenamiento y reducir la varianza cuando el problema es su generalización, es decir, su adaptación a los sets de prueba.

Estimación de Varianza y Bias

El valor del bias se estima como la diferencia entre el error cometido en el set de entrenamiento y el error mínimo posible (lo que se conoce como el error humano y que es calcula por estimación del error de Bayes), Bias=Etrain−Ehuman\textit{Bias}=E_\textrm{train}-E_\textrm{human}Bias=Etrain​−Ehuman​﻿.

Por otro lado, la varianza se estima como la diferencia entre el error en el set de prueba y el error en el set de entrenamiento, es decir Variance=Etest−Etrain\textrm{Variance}=E_\textrm{test}-E_\textrm{train}Variance=Etest​−Etrain​﻿.

En ambos casos los errores se calculan como Error=1−Accuracy\textrm{Error}=1-\textrm{Accuracy}Error=1−Accuracy﻿.

Mejora del rendimiento de un modelo

Lo hiperparámetros, no obstante, deben ajustarse de forma manual antes de comenzar el proceso de entrenamiento, siendo algunos de los más importantes:

  • El número de capas LLL﻿
  • El número de unidades neuronales en cada capa oculta nh[l]n^{[l]}_hnh[l]​﻿
  • Las funciones de activación g[l](z)g^{[l]}(z)g[l](z)﻿
  • La elección de las funciones de pérdida y coste L(y,y′)\mathcal L(y,y')L(y,y′)﻿, J(w,b)J(w,b)J(w,b)﻿
  • El número de iteraciones Epochs\textrm{Epochs}Epochs﻿
  • La tasa de aprendizaje α\alphaα﻿
  • El tamaño del mini-batch mbm_bmb​﻿

Y también otros como la elección del tipo de inicialización, de regularización (siguiente tema) o incluso la elección de un método de optimización para algunos hiperparámetros numéricos.

En líneas generales hay dos métodos para aumentar la eficacia de un modelo:

  • Aumentar la complejidad de éste, si bien está limitado por la potencia computacional y el tiempo disponibles
  • Emplear más datos, lo que de nuevo sufre una limitación por la disponibilidad de éstos

La estrategia a seguir para mejorar el rendimiento consiste en ajustar el equilibrio entre bias y varianza de forma iterativa hasta converger a la eficacia deseada. Para ello cabe preguntarse antes de cada nuevo entrenamiento de la red cuál de los dos conviene favorecer (lo que implicará un decremento del otro).

Algunas técnicas para reducir el bias son:

  • Añadir complejidad (más neuronas, más capas...)
  • Entrenar por más tiempo
  • Emplear un mejor algoritmo de optimización de hiperparámetros
  • Variar la arquitectura

Mientras que para reducir la varianza:

  • Emplear más datos
  • Uso de regularización
  • Variar la arquitectura

Elección de hiperparámetros básicos

Estructura de la red

El problema de elección de la estructura de la red consiste en figurarse cuántas capas ocultas y neuronas por cada una de éstas hacen falta, pues las capas de entrada y salida son obligatorias (para tener una red se necesitan al menos estas dos capas) y el número de neuronas en cada una es una restricción de la topología del problema:

  • La capa de entrada tendrá una neurona por cada atributo
  • La capa de salida tendrá una neurona por cada clase (numérica o binaria)

La elección es, en general, arbitraria, si bien se asume que hay pocas neuronas si el bias es muy elevado y demasiadas si la varianza o el tiempo de entrenamiento lo son.

Empíricamente, se ha comprobado que el uso de estructuras “piramidales” para la capa oculta, comenzando con un número grande de neuronas en la primera y terminando con un número reducido en la última, favorece el rendimiento del modelo.
