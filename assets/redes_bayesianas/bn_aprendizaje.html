Aprendizaje de BN a partir de datos: descubriendo asociación

El proceso de aprendizaje para las redes bayesianas se divide en dos partes, la referida al aprendizaje de la estructura de la red (el conjunto de arcos entre nodos y sus direcciones) y la que trata del aprendizaje de los parámetros de la red (las tablas de probabilidad condicional en el caso de las BN estáticas).

El objetivo es, a partir de un set de datos embruto, obtener una representación en forma de red bayesiana de la información que contienen que sea:

  • Condensada: que resuma lo esencial de los datos reduciendo al máximo el número de parámetros necesarios para especificar la JPD
  • Abstracta: un modelo que describe la JPD que se extrae de los datos
  • Útil: que permita actúar con diferentes tipos de razonamiento

Estimación a partir de datos incompletos

Hasta ahora habíamos asumido que los datos observados eran completos, es decir, incluían el estado de todas las variables de la red a cada entrada de la base de datos, pero en el mundo real a menudo hay que trabajar con bases de datos incompletas.

La opción más sencilla para trabajar con una base de datos incompletos es transformarla en una de datos completos ignorando aquellas entradas a las que le falta el valor de alguna variable (figura de la izquierda), o, incluso, de forma un poco más sofisticada, ignorando sólo los casos que tengan datos incompletos para la estimación de una variable en concreto (figuras de la derecha)

No obstante, en ambos casos se está perdiendo información por el camino, dado que no se está utilizando el total de los datos de los que se dispone.

Otra opción es la inyección de datos faltantes, produciéndolos aleatoriamente (imputation of missing data). Este proceso se suele llevar a cabo considerando la moda de la variable o, alternativamente, la moda de una variable condicionada con la instanciación del resto.

Algoritmo de Esperanza-Maximización (EM)

Es un algoritmo iterativo para aprendizaje a partir de datos incompletos que se basa en dos pasos:

  • Paso de expectación: Se estiman los datos faltantes a partir de sus valores esperados, que se obtienen de la estimación actual de los parámetros, asignando a cada completación un peso empleando un algoritmo de inferencia (o, en la iteración 0, aleatoriamente)
  • Paso de maximización: Usando los datos generados para completar la base de datos, se recalculan los parámetros por el método de máxima verosimilitud

Aprendizaje de estructuras

En general los métodos de aprendizaje de estructura siguen tres líneas:

  • Detección de independencias condicionales (restricciones): Primero se lleva a cabo un estudio de la dependencia o independencia entre las variables ejecutando tests estadísticos para después intentar encontrar la estructura que represente el máximo número de estas relaciones
  • Puntuación+Búsqueda: Se procura encontrar la estructura que mejor encaja con los datos, y requieren de:
    • Una puntuación dada por una métrica o una función de evaluación para medir la aptitud de cada estructura candidato
    • Una estrategia de búsqueda (heurística) para explorar de forma inteligente el espacio de posibles soluciones para la estructura
  • Métodos híbridos: Se guían por técnicas de los dos tipos

Meétodos por restricciones: tests de independencia condicional

Para deducir las relaciones de independencia condicional se puede usar:

  • Una lista de independencias condicionales que ya se conozcan para deducir nuevas a partir de ellas
  • Una JPD de la que se puedan comprobar las relaciones de independencia condicional
  • Una base de datos de casos sobre la que se puedan testar las independencias condicionales

Los métodos para llevar a cabo esta tarea difieren fundamentalmente en:

  • El tipo de información que requieren (orden de las variables...)
  • El tipo de estructura que devuelven (árbol, poliarbol, grafo multiplemente conexo...)
  • El nivel de eficiencia y de garantía que pueden ofrecer
  • Su robustez frente a muestras pequeñas

A la hora de comprobar independencias condicionales, tener las variables en una ordenación σ\sigmaσ﻿ adecuada, puede simplificar en gran manera el problema de aprendizaje de la estructura, tanto para métodos de búsqueda y evaluación como para aquellos que se basan en aprendizaje de CI.

En general, las mejores ordenaciones de variables son las que respetan las relaciones causales conocidas, es decir, incluyen las variables de causa antes que las de efecto. Grafo acíclico parcialmente dirigido (PDAG)

Usando tests de independencia condicional no es posible obtener un único DAG, sino que se obtienen grafos parcialmente dirigidos. Cada PDAG representa una clase de equivalencia de DAGs, de modo que los arcos en un PDAG aparecerán en todos los DAG asociados a su clase de equivalencia, y las aristas del PDAG aparecerán orientadas de diferente manera en cada DAG asociado a su clase (OJO, sin dar lugar a patrones cara a cara).

El algoritmo PC (Spirtes et al. 1993)

Es un algoritmo de aprendizaje de estructuras basado en restricciones cuya idea general consiste en generar un esqueleto para el grafo derivado a través de tests estadísticos para detectar independencias condicionales.

El algoritmo comienza con un grafo completamente conexo y no dirigido, y después procede recurrentemente con tests de independencia condicional para ir eliminando aristas, siendo la salida del algoritmo un PDAG en el que las aristas tienen que ser transformadas en arcos.

Algoritmos de búsqueda local

Dado un grafo candidato, como las métricas más usuales se descomponen muy bien, es útil considerar las operaciones locales de inserción, eliminación e inversión de arcos para generar variantes que sólo difieran localmente del candidato.

Para ello se pueden utilizar algoritmos de búsqueda genética (Larrañaga et al., 1996), de modo que cada individuo del algoritmo genético representa una estructura DAG en forma binaria. No obstante, los operadores de cruce por corte y de mutación por inversión de bit no garantizan que los descendientes satisfagan la condición de DAG, con lo que hay que introducir operadores de reparación.

Métodos híbridos

Combinan la detección de independencias condicionales con los métodos de búsqueda y puntuación. Dos ejemplos son:

  • Obtener diferentes soluciones mediante la aplicación del algoritmo PC (método por restricciones) para después realizar una búsqueda local partiendo de cada una de estas soluciones con una puntuación para evaluar
  • Buscar la estructura que mejor representa las independencias condicionales de los datos y utilizar como puntuación a optimizar el número de éstas que están representadas en la estructrura y comprobadas en la base de datos
